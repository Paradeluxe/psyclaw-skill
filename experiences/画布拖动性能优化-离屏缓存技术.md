# 画布拖动性能优化 - 离屏缓存技术

## 问题描述

画布在拖动时出现卡顿和掉帧，主要原因是每次鼠标移动都触发了完整的重绘流程，包括：
- 遍历所有 connections 计算 loop 内容
- 遍历所有 linePoints 检查连接状态
- 绘制所有 routineRects
- 大量 Canvas API 调用

## 解决方案

使用**离屏 Canvas 缓存技术**，在拖动时只复制缓存图像，而不是重新渲染所有内容。

## 核心实现

### 1. 创建离屏 Canvas

```javascript
// 离屏缓存画布 - 用于优化拖动性能
const offscreenCanvas = document.createElement('canvas');
const offscreenCtx = offscreenCanvas.getContext('2d');
let offscreenCacheValid = false;
let offscreenScale = 1;
let offscreenViewportX = 0;
let offscreenViewportY = 0;
```

### 2. 渲染到离屏缓存

在开始拖动时，将当前完整画面渲染到离屏 canvas：

```javascript
function renderToOffscreen() {
    // 调整离屏画布大小（考虑 DPR）
    if (offscreenCanvas.width !== canvas.width || offscreenCanvas.height !== canvas.height) {
        offscreenCanvas.width = canvas.width;
        offscreenCanvas.height = canvas.height;
    }
    
    offscreenCtx.save();
    offscreenCtx.setTransform(1, 0, 0, 1, 0, 0);
    offscreenCtx.clearRect(0, 0, offscreenCanvas.width, offscreenCanvas.height);
    offscreenCtx.restore();
    
    // 应用 DPR 变换，与主 canvas 保持一致
    offscreenCtx.save();
    offscreenCtx.setTransform(dpr, 0, 0, dpr, 0, 0);
    
    offscreenCtx.save();
    offscreenCtx.translate(viewportX * scale, viewportY * scale);
    offscreenCtx.scale(scale, scale);
    
    // 绘制所有内容：基础线条、连接线、点、routine 块等
    // ... 绘制逻辑
    
    offscreenCtx.restore();
    offscreenCtx.restore();
    
    // 更新缓存状态
    offscreenCacheValid = true;
    offscreenScale = scale;
    offscreenViewportX = viewportX;
    offscreenViewportY = viewportY;
}
```

### 3. 拖动时快速绘制

在 `draw()` 函数中检测如果是平移操作且缓存有效，直接绘制缓存：

```javascript
function draw() {
    // 如果是平移操作且缓存有效，直接绘制缓存
    if (isCanvasPanning && offscreenCacheValid && offscreenScale === scale) {
        ctx.save();
        ctx.setTransform(1, 0, 0, 1, 0, 0);
        ctx.clearRect(0, 0, canvas.width, canvas.height);
        
        // 应用 DPR 变换
        ctx.save();
        ctx.setTransform(dpr, 0, 0, dpr, 0, 0);
        
        // 计算平移偏移量（考虑 DPR）
        const deltaX = (viewportX - offscreenViewportX) * scale;
        const deltaY = (viewportY - offscreenViewportY) * scale;
        
        ctx.drawImage(offscreenCanvas, deltaX, deltaY);
        ctx.restore();
        ctx.restore();
        return;
    }
    
    // 正常绘制流程...
}
```

### 4. 缓存失效处理

在以下情况需要使缓存失效：

```javascript
// 平移结束时清除缓存
canvas.addEventListener('mouseup', (e) => {
    if (isCanvasPanning) {
        isCanvasPanning = false;
        canvas.style.cursor = 'grab';
        offscreenCacheValid = false;  // 清除缓存
        return;
    }
});

// 缩放时清除缓存
function zoomIn() {
    // ... 缩放逻辑
    offscreenCacheValid = false;
    draw();
}

// undo/redo 时清除缓存
function undo() {
    // ... undo 逻辑
    offscreenCacheValid = false;
    draw();
}

// 窗口大小改变时清除缓存
window.addEventListener('resize', () => {
    // ... resize 逻辑
    offscreenCacheValid = false;
    draw();
});
```

## 关键技术点

### 1. DPR (Device Pixel Ratio) 处理

离屏 canvas 必须应用与主 canvas 相同的 DPR 变换，否则在高 DPI 屏幕上会出现内容不完整或模糊的问题：

```javascript
// 主 canvas 设置
canvas.width = window.innerWidth * dpr;
canvas.height = window.innerHeight * dpr;
ctx.setTransform(dpr, 0, 0, dpr, 0, 0);

// 离屏 canvas 同样需要
offscreenCtx.setTransform(dpr, 0, 0, dpr, 0, 0);
```

### 2. Context 切换技巧

由于现有绘制函数都使用全局 `ctx`，在离屏渲染时需要临时切换：

```javascript
// 将 ctx 从 const 改为 let
let ctx = canvas.getContext('2d');

// 临时切换到离屏 context
const savedCtx = ctx;
ctx = offscreenCtx;
drawConnectionLines();  // 这些函数内部使用 ctx
drawPreviewLine();
ctx = savedCtx;  // 恢复
```

### 3. 平移偏移量计算

缓存图像的 viewport 位置与当前 viewport 位置的差值，乘以缩放比例：

```javascript
const deltaX = (viewportX - offscreenViewportX) * scale;
const deltaY = (viewportY - offscreenViewportY) * scale;
ctx.drawImage(offscreenCanvas, deltaX, deltaY);
```

## 性能提升

- **拖动前**：每次鼠标移动都执行完整的渲染流程（O(n) 复杂度，大量 Canvas API 调用）
- **拖动后**：每次鼠标移动只执行一次图像复制操作（GPU 加速，几乎零开销）
- **实际效果**：拖动从卡顿掉帧变为流畅 60fps

## 适用场景

此技术适用于：
- 复杂图形的拖动/平移操作
- 内容不变化只改变 viewport 的场景
- 需要频繁重绘但内容静态的场景

不适用场景：
- 拖动过程中内容需要实时变化
- 需要与拖动内容交互的场景
